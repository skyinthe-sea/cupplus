# 회원 등록 기획서 (상세)

---

## 진입점

| 위치 | 동작 |
|---|---|
| 홈 → 빠른 액션 [회원등록] | 바텀시트 시작 → 풀스크린 확장 |
| 마이탭 → 내 회원 관리 → [+ 등록] | 바텀시트 시작 → 풀스크린 확장 |

---

## 전체 플로우

```
바텀시트 올라옴 (200ms, easeOut)
  → 풀스크린으로 확장 (300ms, easeInOut)
    → Step 1: 기본 정보
      → Step 2: 직업/학력
        → Step 3: 신체/외모
          → Step 4: 성격/취미
            → Step 5: 동의 및 완료
              → [등록 완료] 탭
                → 성공 애니메이션 화면
                  → [프로필 확인] 또는 [홈으로]
```

---

## 공통 UI 요소

### 진행바 (돈트 인디케이터)
```
Step 1:  ● ○ ○ ○ ○
Step 2:  ● ● ○ ○ ○
Step 3:  ● ● ● ○ ○
Step 4:  ● ● ● ● ○
Step 5:  ● ● ● ● ●
```
- **현재/완료 스텝**: Primary 컬러 채움 (#2D5A8E)
- **남은 스텝**: 회색 outline
- 돈트 간 간격: 12.r
- 돈트 크기: 8.r (현재 스텝은 10.r로 약간 크게)
- **애니메이션**: 돈트 채움 시 scale bounce (1.0 → 1.3 → 1.0, 200ms)

### 스텝 헤더
```
┌─────────────────────────────┐
│ ✕                           │  ← 닫기 버튼 (좌상단)
│                             │
│  ● ● ● ○ ○                 │  ← 돈트 인디케이터 (중앙)
│                             │
│  Step 3/5                   │  ← 스텝 번호 (중앙, 서브텍스트)
│  신체/외모 정보              │  ← 스텝 제목 (중앙, 타이틀)
│                             │
```

### 하단 버튼 영역
```
│                             │
│  [이전]          [다음]     │  ← Step 2~4
│                             │
│                 [다음]      │  ← Step 1 (이전 없음)
│                             │
│  [이전]      [등록 완료]    │  ← Step 5
│                             │
```
- **다음 버튼**: Primary 컬러, 필수 필드 미입력 시 비활성화 (회색)
- **이전 버튼**: Outline 스타일
- **등록 완료 버튼**: Primary 컬러, 필수 동의 미체크 시 비활성화

### 스텝 전환 애니메이션
- **다음**: 현재 카드 좌로 슬라이드아웃 + 페이드아웃 → 새 카드 우측에서 슬라이드인 + 페이드인
- **이전**: 현재 카드 우로 슬라이드아웃 + 페이드아웃 → 새 카드 좌측에서 슬라이드인 + 페이드인
- **Duration**: 300ms
- **Curve**: easeInOutCubic
- **스와이프 제스처**: 좌우 스와이프로도 이동 가능 (DragGesture threshold: 100px)

### 닫기/뒤로가기 동작
- ✕ 버튼 또는 시스템 뒤로가기 → **확인 다이얼로그**:
  ```
  ┌─────────────────────────┐
  │                         │
  │  입력 중인 정보가       │
  │  있습니다.              │
  │                         │
  │  나가시겠습니까?        │
  │  (임시저장됩니다)       │
  │                         │
  │  [취소]     [나가기]    │
  └─────────────────────────┘
  ```
- "나가기" 선택 시: 로컬 임시저장 + 화면 닫기
- 다시 진입 시: "작성 중인 회원 정보가 있습니다. 이어서 작성하시겠습니까?" 다이얼로그

### 중간 저장
- **저장 위치**: SharedPreferences (JSON 직렬화)
- **저장 시점**: 각 스텝 이동 시 자동 저장 + 닫기 시
- **만료**: 7일 후 자동 삭제
- **키**: `draft_client_registration_{manager_id}`

### 필드 상세

| 필드 | 필수 | 타입 | 유효성 검사 |
|---|---|---|---|
| 이름 | ✅ | TextField | 2~20자, 한글/영문만 |
| 성별 | ✅ | ToggleButton (2개) | 필수 선택 |
| 생년월일 | ✅ | CupertinoDatePicker | 만 19세 이상 ~ 만 60세 이하 |
| 핸드폰 | ❌ | TextField (number) | 010-XXXX-XXXX 형식, 자동 하이픈 |
| 이메일 | ❌ | TextField (email) | 이메일 형식 검증 |

### 이름 필드
- 포커스 시: 밑줄 Primary 컬러로 변경 (300ms 페이드)
- 입력 시: 실시간 유효성 검사 (2자 미만이면 빨간 경고 텍스트)
- 키보드: 한글/영문 키보드

### 성별 토글
- 2개 토글 버튼 가로 배치
- 미선택: 회색 outline, 연회색 배경
- 선택: Primary 컬러 배경 (#2D5A8E), 흰색 텍스트
- **애니메이션**: 선택 시 배경 컬러 페이드 (200ms) + 미세한 scale bounce

### 생년월일 Cupertino 피커
- 필드 탭 → 바텀시트로 휘일 피커 표시
- 기본값: 1995년 (30세 기준)
- 범위: 1966년 ~ 2007년 (만 19~60세)
- 년/월/일 3개 휘일
- 선택 완료 → 필드에 "1995년 03월 15일" 표시

### 연락처 필드
- 핸드폰: 숫자 키패드, 자동 하이픈 포맷팅 (010-0000-0000)
- 이메일: 이메일 키보드 (@. 포함)
- 둘 다 선택이므로 빈 값 허용

### 다음 버튼 활성화 조건
- 이름 (2자 이상) + 성별 선택 + 생년월일 선택 = **활성화**

### 필드 상세

| 필드 | 필수 | 타입 | 비고 |
|---|---|---|---|
| 직업 | ✅ | TextField + 자동완성 | 미리 정의된 직업 목록에서 자동완성, 없으면 자유 입력 |
| 회사 | ❌ | TextField | 자유 입력 |
| 학력 수준 | ❌ | ChoiceChip (단일) | 고졸/전문대/대졸/석사/박사 |
| 학교명 | ❌ | TextField + 자동완성 | 한국 대학교 목록 기반 자동완성 |
| 전공 | ❌ | TextField | 자유 입력 |
| 연소득대 | ❌ | RadioButton | 6개 범위 중 택1 |

### 자동완성 동작
- 2글자 이상 입력 시 드롭다운 표시
- 매칭 항목이 없으면 입력값 그대로 사용
- 드롭다운 항목 탭 → 필드에 채움 + 키보드 닫기
- **애니메이션**: 드롭다운 슬라이드다운 (200ms, easeOut)

### 학력 수준 칩
- 가로 스크롤 가능한 ChoiceChip 그룹
- 미선택: outline 스타일
- 선택: Primary 컬러 filled
- 선택 시 bounce 애니메이션 (100ms)

### 연소득대 라디오
- 세로 리스트, 라디오 버튼
- 선택 시 라디오 아이콘 채움 + 해당 행 배경 연한 Primary 컬러
- 숫자는 원 단위로 컴마 포맷팅

### 다음 버튼 활성화 조건
- 직업 (1자 이상) = **활성화** (나머지 모두 선택)

│  │  +  ││ 📷1 ││ 📷2 │     │  ← 사진 그리드

│  첫 번째 사진이 대표 사진   │

│                             │
│  [이전]          [다음]     │
└─────────────────────────────┘
```

### 필드 상세

| 필드 | 필수 | 타입 | 비고 |
|---|---|---|---|
| 키 | ✅ | CupertinoPicker (인라인) | 140~200cm, 기본 170cm |
| 체형 | ❌ | ChoiceChip (단일) | 마른/약간마른/보통/약간통통/통통 |
| 프로필 사진 | ❌ | 이미지 업로드 | 최대 5장, 카메라+갤러리 |

### 키 휘일 피커
- **인라인**: 바텀시트 없이 화면에 직접 표시
- 범위: 140cm ~ 200cm
- 기본값: 170cm (남) / 160cm (여) — Step 1의 성별에 따라
- 선택된 값 아래에 크게 "166 cm" 표시
- **햅틱 피드백**: 값 변경 시 가벼운 진동

### 체형 칩
- Wrap 레이아웃 (2줄)
- 선택 스타일: Step 2 학력 칩과 동일

### 프로필 사진 업로드
- **그리드 레이아웃**: 3열, 정사각형 셀
- **첫 번째 셀**: [+추가] 버튼 (점선 테두리, 아이콘)
- **업로드된 사진**: 썸네일 표시 + 우상단 ✕ 삭제 버튼
- **첫 번째 사진**: "대표" 뱃지 표시
- **드래그 앤 드롭**: 사진 순서 변경 가능 (롱프레스 후 드래그)

#### 사진 업로드 플로우
```
[+추가] 탭
  → 바텀시트: [카메라로 촬영] [갤러리에서 선택]
    → 카메라: 촬영 → 크롭 화면 (1:1 비율)
    → 갤러리: 선택 → 크롭 화면 (1:1 비율)
      → 이미지 압축 (80%, 최대 1024x1024)
      → EXIF 자동 제거
      → 그리드에 썸네일 표시
```

#### 사진 업로드 애니메이션
- 업로드 중: 셀에 원형 프로그레스 인디케이터
- 완료: 프로그레스 → 체크마크 → 페이드아웃 → 썸네일 페이드인 (300ms)
- 삭제: 셀 축소 (scale 1.0 → 0, 200ms) → 다른 셀 이동 애니메이션

#### 사진 저장
- 등록 완료 전까지: 로컬 임시 파일로 보관
- 등록 완료 시: Supabase Storage `profile-photos/{client_id}/` 에 업로드

### 다음 버튼 활성화 조건
- 키 선택 = **활성화** (체형, 사진 선택)

│                             │
│  [이전]          [다음]     │
└─────────────────────────────┘
```

### 필드 상세

| 필드 | 필수 | 타입 | 비고 |
|---|---|---|---|
| 종교 | ✅ | ChoiceChip (단일) | 무교/기독교/천주교/불교/기타 |
| 취미 | ❌ | FilterChip (복수) + 자유 입력 | 최대 5개, 20개 기본 태그 + 직접 입력 |
| 자기소개 | ❌ | TextField (멀티라인) | 최대 300자, 실시간 글자 수 표시 |

### 종교 칩
- 단일 선택 ChoiceChip
- "기타" 선택 시: 하단에 텍스트 필드 나타남 (slideDown, 200ms)

### 취미 태그
- **기본 태그 20개**: Wrap 레이아웃
  - 여행, 등산, 요가, 운동, 독서, 요리, 카페, 영화, 음악, 게임
  - 사진, 그림, 맛집탐방, 캠핑, 낚시, 필라테스, 미술관, 볼링, 코딩, 와인
- **선택 스타일**: outline → Primary filled + 흰색 텍스트
- **선택 제한**: 5개 초과 선택 시 진동 + "최대 5개까지 선택 가능합니다" 스낵바
- **선택 애니메이션**: 칩 scale bounce (1.0 → 1.1 → 1.0, 150ms)

### 직접 입력 태그
- [+ 직접 입력] 탭 → 텍스트 필드 + [추가] 버튼 표시 (slideDown, 200ms)
- 입력 후 [추가] 탭 → 태그 추가 + 텍스트 필드 클리어
- 최대 10자, 중복 방지
- 추가된 커스텀 태그는 기본 태그와 동일한 스타일

### 선택된 취미 영역
- 태그 옆에 ✕ 버튼 (탭하면 선택 해제)
- "3/5" 카운트 표시
- 태그 추가/제거 시 **AnimatedList** 전환

### 자기소개
- 멀티라인 TextField (최소 3줄)
- 플레이스홀더: "밝고 긍정적인 성격으로, 주말엔 카페에서 책 읽는 것을 좋아합니다..."
- 실시간 글자 수 카운터 (우하단)
- 300자 초과 시 입력 차단 + 카운터 빨간색

### 다음 버튼 활성화 조건
- 종교 선택 = **활성화** (취미, 자기소개 선택)

## Step 5: 동의 및 완료

### 레이아웃
```
┌─────────────────────────────┐
│ ✕                           │
│        ● ● ● ● ●            │
│        Step 5/5             │
│     동의 및 완료             │
│                             │
│  등록 정보를 확인하고        │
│  약관에 동의해주세요.        │
│                             │
│  ┌─────────────────────┐    │
│  │ ☐ 전체 동의          │    │  ← 전체 동의 체크
│  ├─────────────────────┤    │
│  │ ☐ 서비스 이용 약관    │    │
│  │   (필수)    [보기>]  │    │
│  │                      │    │
│  │ ☐ 개인정보 수집/이용  │    │
│  │   (필수)    [보기>]  │    │
│  │                      │    │
│  │ ☐ 마케팅 정보 수신    │    │
│  │   (선택)    [보기>]  │    │
│  └─────────────────────┘    │
│                             │
│  [이전]      [등록 완료]    │
└─────────────────────────────┘
```

### 동의 항목

| 항목 | 필수 | 설명 |
|---|---|---|
| 서비스 이용 약관 | ✅ | CupPlus 매칭 서비스 이용 약관 |
| 개인정보 수집/이용 | ✅ | 회원 정보 수집 및 제3자 제공 동의 |
| 마케팅 정보 수신 | ❌ | 프로모션, 이벤트 알림 동의 |

### 전체 동의 체크
- 전체 동의 체크 → 3개 모두 체크
- 전체 동의 해제 → 3개 모두 해제
- 개별 해제 시: 전체 동의 자동 해제
- 3개 모두 체크 시: 전체 동의 자동 체크

### [보기>] 탭 동작
- 바텀시트로 약관 전문 표시 (스크롤 가능)
- 약관 텍스트: 마크다운 또는 플레인 텍스트
- 하단 [확인] 버튼으로 닫기

### 체크박스 애니메이션
- 체크 시: ☐ → ☑ 스케일 bounce (1.0 → 1.2 → 1.0, 150ms) + Primary 컬러 채움
- 전체 동의 시: 개별 체크박스들이 위에서 아래로 순차적 체크 (각 50ms 딜레이)

### 등록 완료 버튼 활성화 조건
- 필수 2개 동의 (서비스 이용 약관 + 개인정보 수집) = **활성화**

### 등록 완료 탭 시 동작
```
[등록 완료] 탭
  → 버튼 로딩 스피너 (원형 프로그레스)
  → Supabase clients INSERT
  → 사진 업로드 (있으면)
  → contract_agreements INSERT (동의 기록)
  → 성공 → 성공 화면 전환
  → 실패 → 에러 스낵바 + 버튼 복원
```

## 성공 화면

### 레이아웃
```
┌─────────────────────────────┐
│                             │
│                             │
│                             │
│          ┌───┐              │
│          │ ✓ │              │  ← 체크마크 스케일업 애니메이션
│          └───┘              │
│                             │
│     회원 등록 완료!          │  ← 페이드인 (딜레이 300ms)
│                             │
│   김서연님이                 │  ← 페이드인 (딜레이 500ms)
│   등록되었습니다             │
│                             │
│                             │
│  [프로필 확인]  [홈으로]     │  ← 페이드인 (딜레이 700ms)
│                             │
└─────────────────────────────┘
```

### 애니메이션 시퀀스
1. **0ms**: 화면 전환 (페이드)
2. **100ms**: 체크마크 아이콘 등장
   - scale: 0 → 1.3 → 1.0 (spring curve, 400ms)
   - 컬러: Primary → Green (#2E7D32)
   - 크기: 80.r
3. **300ms**: "회원 등록 완료!" 텍스트 페이드인 + slideUp (20px)
4. **500ms**: 회원 이름 텍스트 페이드인 + slideUp
5. **700ms**: 버튼 2개 페이드인 + slideUp
6. **100ms**: 햅틱 피드백 (성공 진동)

### 버튼 동작
- **[프로필 확인]**: 등록된 회원의 상세 프로필 화면으로 이동
- **[홈으로]**: 홈 화면으로 이동 (화면 스택 초기화)

## 데이터 저장

### clients 테이블 INSERT
```sql
INSERT INTO clients (
  manager_id,      -- 현재 매니저 ID
  region_id,       -- 현재 매니저의 region_id
  full_name,       -- Step 1: 이름
  gender,          -- Step 1: 성별 ('F' | 'M')
  birth_date,      -- Step 1: 생년월일
  phone,           -- Step 1: 핸드폰 (nullable)
  email,           -- Step 1: 이메일 (nullable)
  occupation,      -- Step 2: 직업
  company,         -- Step 2: 회사 (nullable)
  education_level, -- Step 2: 학력수준 (nullable)
  school,          -- Step 2: 학교명 (nullable)
  major,           -- Step 2: 전공 (nullable)
  annual_income_range, -- Step 2: 연소득대 (nullable)
  height_cm,       -- Step 3: 키
  body_type,       -- Step 3: 체형 (nullable)
  religion,        -- Step 4: 종교
  hobbies,         -- Step 4: 취미 (text[] 배열, nullable)
  bio,             -- Step 4: 자기소개 (nullable)
  status           -- 'active'
) VALUES (...);
```

### 스키마 변경 필요 사항
기존 `clients` 테이블에 추가 필요한 컬럼:
- `phone TEXT`  (이미 있음)
- `email TEXT` (신규)
- `education_level TEXT` (신규 — 'high_school' | 'associate' | 'bachelor' | 'master' | 'doctorate')
- `school TEXT` (신규)
- `major TEXT` (신규)
- `body_type TEXT` (신규 — 'slim' | 'slightly_slim' | 'average' | 'slightly_chubby' | 'chubby')
- `hobbies TEXT[]` (신규)
- `bio TEXT` (신규)

## 에러 처리

| 상황 | 처리 |
|---|---|
| 네트워크 오류 | "네트워크 연결을 확인해주세요" 스낵바 + 재시도 버튼 |
| 서버 오류 | "등록에 실패했습니다. 다시 시도해주세요" 스낵바 |
| 사진 업로드 실패 | "사진 업로드에 실패했습니다" + 회원은 등록 완료, 사진은 나중에 재업로드 |
| 중복 회원 (동명이인) | 중복 방지 없음 — 동명이인 허용 |
| 임시저장 복원 실패 | 무시하고 빈 폼으로 시작 |
