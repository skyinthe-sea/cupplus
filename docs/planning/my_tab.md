## 4-1. 프로필 카드

### 구성
- 프로필 사진 (또는 이름 아바타)
- 매니저 이름 (닉네임 우선)
- 소속 정보 (region 기반)
- [프로필 수정] 버튼

### 매니저 인증 뱃지
- 인증 완료 시: 프로필 이름 옆에 **녹색 ✅ 뱃지** (작게)
- 미인증 시: 뱃지 없음
- 인증 대기 중: 회색 시계 아이콘 (⏳) 또는 "인증 대기중" 텍스트

### 프로필 수정 화면
- 닉네임 수정 (기존 구현 활용)
- 프로필 사진 변경
- 연락처 수정
- 연동 계정 관리 (Google/Kakao)

## 4-2. 내 회원 관리

### 진입
- "내 회원 관리" 탭 → 풀스크린 회원 목록 화면으로 이동

### 회원 목록 화면
```
┌───────────────────────────────┐
│ ← 내 회원 관리         [+ 등록] │
├───────────────────────────────┤
│ 🔍 이름 검색                   │
├───────────────────────────────┤
│ [전체 5] [활성 3] [휴지 1] [매칭 1] │  ← 상태 탭
├───────────────────────────────┤
│ ┌───────────────────────────┐ │
│ │ 김서연 29세·여 · 활성     │ │
│ │ 마케팅 매니저 · 네이버    │ │
│ │ 인증: ✅재직 ✅학력       │ │
│ └───────────────────────────┘ │
│ ┌───────────────────────────┐ │
│ │ 이준호 34세·남 · 매칭중   │ │
│ │ ...                       │ │
│ └───────────────────────────┘ │
└───────────────────────────────┘
```

### 상태 탭
| 탭 | 필터 | 설명 |
|---|---|---|
| 전체 | 없음 | 모든 회원 |
| 활성 | status = 'active' | 매칭 가능한 회원 |
| 휴지 | status = 'paused' | 일시 중단 회원 |
| 매칭중 | status = 'matched' | 현재 매칭 진행 중 |

### 회원 카드 탭 → 회원 상세 화면
- 프로필 전체 정보
- 인증 서류 상태
- 매칭 이력
- [수정] [상태변경] [삭제] 버튼

### CRUD 기능

#### 등록 (Create)
- [+ 등록] 버튼 → 회원 등록 폼 (풀스크린)
- 필수: 이름, 성별, 생년월일
- 선택: 직업, 회사, 학력, 키, 종교, 연소득대, 프로필 사진
- 저장 → clients INSERT (manager_id = me, region_id = my region)

#### 조회 (Read)
- 목록: clients WHERE manager_id = me
- 상세: 프로필 전체 + 인증 서류 + 매칭 이력

#### 수정 (Update)
- 회원 상세 → [수정] → 프로필 편집 화면
- 모든 필드 수정 가능
- 상태 변경: active ↔ paused ↔ withdrawn

#### 삭제 (Delete)
- 회원 상세 → [삭제] → 확인 다이얼로그
- **소프트 삭제**: status = 'withdrawn' (DB에서 실제 삭제하지 않음)

#### 삭제 시 사이드이펙트 (실시간 처리)

회원이 삭제(withdrawn)되면 아래 모든 처리가 **실시간**으로 반영되어야 함:

| 영역 | 처리 | 실시간 |
|---|---|---|
| **프로필 마켓** | 목록에서 즉시 사라짐 | Supabase Realtime |
| **좋아요 목록** | 다른 매니저의 좋아요 탭에서 딤 처리 + "삭제된 회원" 표시 | Realtime |
| **대기 매칭** | 해당 회원이 포함된 pending 매칭 → 자동 취소 (status = 'cancelled') | DB Trigger |
| **진행 중 매칭** | accepted/meeting_scheduled 매칭 → 상대 매니저에게 "상대 회원이 탈퇴했습니다" 알림 | Trigger + 알림 |
| **채팅방** | 관련 채팅방에 시스템 메시지: "삭제된 회원입니다" 표시 | Realtime |
| **매칭 상세** | 삭제된 회원 프로필 → "삭제된 회원" 플레이스홀더 표시 | 클라이언트 처리 |
| **알림** | 상대 매니저에게 매칭 취소/회원 탈퇴 알림 발송 | Edge Function |

#### 구현 방식
- **DB Trigger**: clients.status = 'withdrawn' UPDATE 시 자동 실행
  1. 해당 회원의 pending 매칭 전부 cancelled 처리
  2. 진행 중 매칭의 상대 매니저에게 알림 INSERT
  3. 관련 채팅방에 시스템 메시지 INSERT
- **클라이언트**: Supabase Realtime 구독으로 UI 즉시 반영

## 4-3. 구독 관리

### 구성
- 현재 구독 플랜 표시 (Free / Standard / Premium)
- 일일 매칭 사용량: "오늘 1/1건 사용" (Free 기준)
- [플랜 변경] 버튼 → RevenueCat Paywall

### 구독 티어 표시
| 티어 | 일일 매칭 | 표시 |
|---|---|---|
| Free | 1건/일 | "무료 플랜 · 오늘 0/1건" |
| Standard | 10건/일 | "스탠다드 · 오늘 3/10건" |
| Premium | 무제한 | "프리미엄 · 무제한" |

### 데이터 소스
- RevenueCat SDK (`Purchases.getCustomerInfo()`)
- `daily_match_counts` 테이블 (오늘 사용량)

## 4-4. 설정

### 항목
| 설정 | UI | 설명 |
|---|---|---|
| 알림 설정 | 종류별 토글 | 매칭/채팅/인증/시스템 알림 ON/OFF |
| 언어 | 선택 (한국어/English) | 앱 언어 변경 |
| 다크 모드 | 토글 (시스템/라이트/다크) | 테마 모드 |

### 알림 설정 상세
- 매칭 알림: 매칭 요청/수락/거절 알림 (기본 ON)
- 채팅 알림: 새 메시지 알림 (기본 ON)
- 인증 알림: 매니저 인증 승인/반려 (기본 ON)
- 시스템 알림: 공지사항/구독 만료 (기본 ON)
- 데이터: `managers.notification_settings` JSONB 컬럼
- 상세 기획: `docs/planning/push_notification.md` 참조
