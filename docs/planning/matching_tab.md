# 매칭 탭 기획서 (프로필 마켓)

## 역할
- **다른 매니저의 회원**을 탐색하고, 내 회원과 매칭을 생성하는 화면
- 매칭 관리(수락/거절)는 홈 화면의 "대기 매칭" 바텀시트에서 처리

## 레이아웃

```
┌───────────────────────────────┐
│ 프로필 마켓                 ≡  │  ← 제목 + 필터
│ 총 N명의 프로필                │
├───────────────────────────────┤
│ 🔍 이름, 직업, 회사, 지역 검색  │  ← 검색바
├───────────────────────────────┤
│ [전체 12] [여성 6] [남성 6]    │  ← 성별 탭
├───────────────────────────────┤
│ ┌───────────────────────────┐ │
│ │ 👤 김서연 ✅ 31세·여       │ │  ← 프로필 카드
│ │    마케팅 매니저 · 네이버   │ │
│ │    서울대 경영학과          │ │
│ │    [서울 강남] 여행 요가... │ │
│ │    📅 3d ago      ❤️ 5    │ │
│ └───────────────────────────┘ │
│ ┌───────────────────────────┐ │
│ │ 👤 이준호   34세·남        │ │
│ │    ...                    │ │
│ └───────────────────────────┘ │
└───────────────────────────────┘
```

---

## 2-1. 헤더

### 구성
- **좌측**: "프로필 마켓" 타이틀 + "총 N명의 프로필" 서브텍스트
- **우측**: 필터 아이콘 (≡)

---

## 2-2. 검색바

### 검색 대상 필드
- 이름 (full_name)
- 직업 (occupation)
- 회사 (company)
- 지역 (region)

### 동작
- 실시간 검색 (debounce 300ms)
- 검색어가 위 필드 중 하나라도 매칭되면 표시

---

## 2-3. 성별 탭

| 탭 | 필터 | 표시 |
|---|---|---|
| 전체 | 필터 없음 | 전체 수 뱃지 |
| 여성 | gender = 'F' | 여성 수 뱃지 |
| 남성 | gender = 'M' | 남성 수 뱃지 |
| ❤️ 좋아요 | 내가 좋아요 누른 프로필 | 좋아요 수 뱃지 |

### 좋아요 탭 특수 규칙
- 좋아요 누른 회원 중 **매칭 완료된 회원**: 딤(반투명) 처리 + "매칭완료" 태그 표시
  - 매칭 요청 버튼 비활성화
  - 좋아요 해제하면 목록에서 사라짐
- 좋아요 누른 회원 중 **active 회원**: 정상 표시

---

## 2-4. 필터 (바텀시트)

필터 아이콘 탭 → 바텀시트로 필터 옵션 표시

| 필터 | UI | 값 |
|---|---|---|
| 나이대 | RangeSlider | 20~60세 |
| 키 범위 | RangeSlider | 140~200cm |
| 학력 | 드롭다운/칩 | 고졸/전문대/대졸/석사/박사 |
| 직업군 | 멀티선택 칩 | IT/의료/법조/금융/교육/공무원/기타 |
| 종교 | 멀티선택 칩 | 무교/기독교/천주교/불교/기타 |
| 연소득대 | 드롭다운 | 3천만~1억+ (범위) |
| 인증 여부 | 토글 | 인증완료 회원만 보기 |

### 필터 동작
- 적용 버튼으로 일괄 적용
- 활성 필터 개수 표시 (필터 아이콘 옆 뱃지)
- "초기화" 버튼으로 전체 필터 해제

---

## 2-5. 프로필 카드

### 카드 구성 요소
| 요소 | 설명 |
|---|---|
| 프로필 사진 | 원형, 사진 없으면 성 한글자 아바타 |
| 이름 | full_name |
| 인증 뱃지 | ✅ 인증 서류 1개 이상 승인 시 표시 |
| 나이·성별 | "31세 · 여" |
| 직업·회사 | "마케팅 매니저 · 네이버" |
| 학력 | "서울대 경영학과" |
| 지역 태그 | 강조 칩 (Primary 컬러 배경) |
| 취미 태그 | 일반 칩 (최대 3개) |
| 등록일 | "3d ago" 또는 날짜 |
| 하트 수 | ❤️ N — 다른 매니저들이 누른 관심 수 |

### 하트(좋아요) 기능
- **누가**: 매니저가 (다른 매니저의) 회원에게 누름
- **의미**: "이 회원에게 관심 있음" 표시 (매칭 요청 전 단계)
- **동작**: 카드의 하트 아이콘 탭 → 토글 (좋아요/취소)
- **표시**: 해당 회원이 받은 총 하트 수
- **데이터**: 별도 `profile_likes` 테이블 필요 (manager_id, client_id, created_at)

### 카드 탭 동작
- 카드 탭 → **프로필 상세 화면** (풀스크린 push)

---

## 2-6. 프로필 상세 화면

### 구성
- 프로필 사진 (큰 사이즈)
- 기본 정보: 이름, 나이, 성별, 키, 지역
- 직업 정보: 직업, 회사, 연소득대
- 학력 정보: 학교, 전공
- 추가 정보: 종교, 취미, 자기소개, 이상형 메모
- 인증 서류 목록: 인증 완료된 서류 뱃지 표시
- 매칭 이력: 이 회원의 매칭 요청/수락/거절 이력 (선택)

### 하단 버튼
- **[매칭 요청]** 버튼 (Primary)
- **[하트]** 버튼 (Secondary)

---

## 2-7. 매칭 생성 플로우

### 플로우
```
프로필 상세 → [매칭 요청] 탭
  → 내 회원 선택 화면 (바텀시트)
    → 내 회원 목록 (이성만 표시)
      → 회원 선택
        → 확인 다이얼로그 ("김서연 ↔ 이준호 매칭을 요청하시겠습니까?")
          → 매칭 생성 (matches INSERT)
            → 상대 매니저에게 실시간 알림
```

### 핵심 규칙

#### 매니저 인증 검증
- 매칭 요청 시 매니저 인증 상태 체크:
  - **verified**: 정상 진행
  - **pending**: "인증 검토 중입니다. 승인 후 매칭 요청이 가능합니다." 안내 다이얼로그
  - **unverified / rejected**: "매칭 요청을 위해 매니저 인증이 필요합니다." + [인증하기] 버튼 → 인증 서류 제출 플로우 시작
- 서버사이드에서도 인증 상태 체크 (미인증 매니저의 matches INSERT 차단)

#### 성별 검증
- **동일 성별 매칭 불가**: 내가 여성 회원만 있는데 여성 프로필을 보고 있으면 [매칭 요청] 버튼 비활성화
- 버튼 비활성 시 안내 텍스트: "매칭 가능한 이성 회원이 없습니다"
- 서버사이드에서도 동일 성별 매칭 INSERT 차단 (DB constraint 또는 Edge Function)

#### 내 회원 자동 선택
- 이성 회원이 **1명**만 있으면 → 회원 선택 단계 건너뛰고 바로 확인 다이얼로그
- 이성 회원이 **2명 이상**이면 → 회원 선택 바텀시트 표시

#### 실시간 처리
- 매칭 생성 → `matches` 테이블 INSERT
- Supabase Realtime으로 **상대 매니저에게 즉시 알림**
- 상대 매니저의 홈 화면 "대기 매칭" 카운트 실시간 증가
- 상대 매니저의 알림 벨 뱃지 실시간 업데이트

### 매칭 상태 흐름
```
pending → accepted → meeting_scheduled → completed
       ↘ declined
```

### 데이터
- `matches` 테이블 INSERT:
  - client_a_id: 내 회원
  - client_b_id: 상대 회원
  - client_a_region: 내 region_id
  - client_b_region: 상대 region_id
  - manager_id: 나 (요청자)
  - status: 'pending'

---

## 2-8. 데이터 소스

### 프로필 목록
- `clients` 테이블 WHERE manager_id != current_user AND status = 'active' (매칭완료 회원은 제외)
- **좋아요 탭**: `profile_likes` JOIN `clients` WHERE liked_by = current_user (매칭완료 포함, 딤 처리)
- RLS: region_id 기반 (같은 리전 내 회원만 표시)
- 정렬: 토글 버튼으로 선택 가능
  - **최신순** (기본): created_at DESC
  - **좋아요순**: 하트 수 DESC
- 페이지네이션: 20건씩 무한 스크롤

### 프로필 상세
- `clients` 테이블 JOIN `verification_documents` (인증 상태)
- `profile_likes` 테이블 (하트 수)
