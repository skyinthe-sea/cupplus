# 채팅 탭 기획서

## 역할
- **매니저 ↔ 매니저** 간 실시간 채팅
- 매칭이 **수락(성사)되면 자동으로 채팅방 생성**
- 매칭 관련 논의를 위한 소통 채널

---

## 3-1. 채팅 목록 화면

### 레이아웃

```
┌───────────────────────────────┐
│ 채팅                          │
├───────────────────────────────┤
│ ┌───────────────────────────┐ │
│ │ 👤 박매니저               │ │
│ │ 김서연 ↔ 이준호 매칭       │ │  ← 매칭 컨텍스트
│ │ "네, 다음주 화요일 괜..."  │ │  ← 마지막 메시지
│ │              오후 2:30  ② │ │  ← 시간 + 안읽은 수
│ └───────────────────────────┘ │
│ ┌───────────────────────────┐ │
│ │ 👤 이매니저               │ │
│ │ 최민수 ↔ 정하은 매칭       │ │
│ │ "프로필 사진 보내드릴..."  │ │
│ │              어제         │ │
│ └───────────────────────────┘ │
└───────────────────────────────┘
```

### 구성 요소
| 요소 | 설명 |
|---|---|
| 상대 매니저 아바타 | 프로필 사진 또는 이름 아바타 |
| 상대 매니저 이름 | 매니저 닉네임 또는 본명 |
| 매칭 컨텍스트 | "회원A ↔ 회원B 매칭" (어떤 매칭 건인지) |
| 마지막 메시지 | 텍스트 미리보기 (이미지면 "사진" 표시) |
| 시간 | 마지막 메시지 시간 (오늘이면 시간, 아니면 날짜) |
| 안읽은 수 뱃지 | 읽지 않은 메시지 수 (0이면 숨김) |

### 정렬
- 마지막 메시지 시간 DESC (최신 대화 위로)

### 빈 상태
- "아직 채팅이 없습니다. 매칭이 성사되면 자동으로 채팅방이 생성됩니다."

### 데이터 소스
- `conversations` 테이블 WHERE participant_a = me OR participant_b = me
- JOIN `messages` (마지막 메시지)
- JOIN `matches` (매칭 컨텍스트)
- Supabase Realtime: 새 메시지 수신 시 목록 자동 갱신

## 3-2. 채팅방 화면

### 레이아웃

```
┌───────────────────────────────┐
│ ← 박매니저                    │  ← 헤더
│ 김서연 ↔ 이준호 매칭           │  ← 매칭 컨텍스트 바
├───────────────────────────────┤
│         2025년 3월 10일        │  ← 날짜 구분선
│                               │
│    안녕하세요, 매칭 건으로     │  ← 상대 메시지
│    연락드립니다.               │
│                    오후 2:30  │
│                               │
│  네, 확인했습니다!         ✓✓ │  ← 내 메시지 + 읽음
│  오후 2:32                    │
│                               │
│  [사진]                   ✓✓ │  ← 이미지 메시지
│  오후 2:35                    │
│                               │
├───────────────────────────────┤
│ [📷] 메시지를 입력하세요 [전송] │  ← 입력 영역
└───────────────────────────────┘
```

### 헤더
- **뒤로가기** 버튼
- **상대 매니저 이름**
- **매칭 컨텍스트 바**: "김서연 ↔ 이준호 매칭" (탭 시 매칭 상세로 이동)

### 메시지 유형
| 유형 | 설명 |
|---|---|
| 텍스트 | 일반 텍스트 메시지 |
| 이미지 | 사진 전송 (카메라/갤러리에서 선택) |

### 읽음 확인 (카카오톡 스타일)
- 내 메시지: ✓ (전송됨) → ✓✓ (읽음)
- 1:1 채팅이므로 상대가 읽으면 바로 ✓✓
- `read_receipts` 테이블로 관리

### 이미지 전송
- 📷 버튼 → 카메라/갤러리 선택
- 이미지 압축 후 Supabase Storage `chat-images` 버킷에 업로드
- 썸네일 표시 (Supabase Image Transformation)
- 탭 시 풀스크린 이미지 뷰어

### 입력 영역
- 텍스트 입력 필드 (멀티라인)
- 📷 이미지 첨부 버튼
- 전송 버튼 (텍스트 입력 시에만 활성화)

### 실시간 처리
- **Supabase Realtime** `.stream()` 으로 메시지 실시간 수신
- 새 메시지 수신 시 자동 스크롤
- 상대방 입장 시 읽음 상태 실시간 업데이트

### 날짜 구분선
- 메시지 간 날짜가 바뀌면 날짜 구분선 표시
- "오늘", "어제", "2025년 3월 9일" 형식

## 3-3. 채팅방 자동 생성 규칙

### 트리거
- `matches` 테이블에서 status가 `'accepted'`로 변경될 때

### 생성 로직 (서버사이드)
```
매칭 수락 시:
1. conversations INSERT (participant_a = 요청 매니저, participant_b = 수락 매니저)
2. 자동 시스템 메시지: "매칭이 성사되었습니다. 김서연 ↔ 이준호"
3. 양측 매니저에게 알림 (채팅방 생성됨)
```

### 구현 방식
- Supabase Database Trigger 또는 Edge Function
- matches UPDATE → conversations INSERT → system message INSERT

### 1매칭 = 1채팅방
- 동일 매니저 간 여러 매칭이 있어도, 매칭별로 별도 채팅방 생성
- 채팅방에 match_id 연결 (conversations 테이블에 match_id FK 추가 필요)

## 3-4. 데이터 스키마 변경 필요사항

### conversations 테이블 수정
```sql
ALTER TABLE conversations ADD COLUMN match_id UUID REFERENCES matches(id);
```

### profile_likes 테이블 신규 (매칭 탭 좋아요 기능)
```sql
CREATE TABLE profile_likes (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  manager_id UUID REFERENCES managers(id),
  client_id UUID REFERENCES clients(id),
  created_at TIMESTAMPTZ DEFAULT now(),
  UNIQUE(manager_id, client_id)
);
```

## 3-5. 페이지네이션
