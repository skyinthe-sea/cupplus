# 홈 화면 기획서

## 레이아웃 (A안: 피드 중심)

```
┌───────────────────────────────┐
│ cup+              🔔(N)       │  ← 1-1. AppBar
├───────────────────────────────┤
│ 좋은 아침이에요, OOO님         │  ← 1-2. 인사 헤더
├───────────────────────────────┤
│ [회원등록] [매칭생성]           │  ← 1-3. 빠른 액션
├───────────────────────────────┤
│ 오늘의 할일                    │  ← 1-4. 오늘의 할일
│ ⚠ 대기 매칭 3건       [보기>]  │
│ 💬 새 메시지 8건      [보기>]  │
├───────────────────────────────┤
│ 최근 활동                      │  ← 1-5. 활동 피드
│ 10:30 김서연↔이준호 매칭수락 ✅ │
│ 09:15 박지은 신규 회원 등록 👤  │
│ 어제 최민수 프로필 등록         │
└───────────────────────────────┘
```

## 1-1. AppBar

### 구성
- **좌측**: `cup+` 로고 텍스트 (Primary 컬러, Bold)
- **우측**: 알림 벨 아이콘 + 읽지 않은 알림 수 뱃지 (빨간 점 또는 숫자)

### 알림 기능 상세

#### 동작 방식
- 알림 아이콘 탭 → **바텀시트**로 알림 목록 표시
- 각 알림 탭 → 해당 상세 화면으로 이동 (매칭 상세, 채팅방 등)

#### 알림 종류 (MVP)
| 종류 | 트리거 | 예시 텍스트 |
|---|---|---|
| 매칭 응답 | 매칭 수락/거절 시 | "김서연↔이준호 매칭이 수락되었습니다" |
| 새 메시지 | 채팅 메시지 수신 시 | "박매니저님이 메시지를 보냈습니다" |
| 매니저 인증 | 인증 승인/반려 시 | "매니저 인증이 승인되었습니다" |
| 시스템 알림 | 공지사항, 구독 만료 등 | "구독이 3일 후 만료됩니다" |

#### 읽음 처리
- **자동 읽음**: 바텀시트를 열면 보이는 알림 자동 읽음 처리
- 뱃지 숫자는 읽지 않은 알림 수 반영 (실시간)

#### 데이터 소스
- `notifications` 테이블 (status = 'pending' | 'sent')
- 실시간 업데이트: Supabase Realtime 구독

## 1-2. 인사 헤더

### 구성
- 시간대별 인사 + 매니저 이름(닉네임 우선)

### 시간대별 인사 규칙
| 시간 | 인사 |
|---|---|
| 06:00~11:59 | "좋은 아침이에요, OOO님" |
| 12:00~17:59 | "좋은 오후예요, OOO님" |
| 18:00~21:59 | "좋은 저녁이에요, OOO님" |
| 22:00~05:59 | "늦은 시간 고생하세요, OOO님" |

### 이름 표시 우선순위
1. 닉네임 (managers.nickname)
2. 본명 (managers.full_name)
3. 이메일 앞부분
4. 'User' (폴백)

## 1-3. 빠른 액션

### 구성
가로 스크롤 칩/버튼 2개:
- **회원등록** — 바텀시트로 간편 등록 폼
- **매칭생성** — 오늘의 추천 프로필 바텀시트

### [회원등록] 바텀시트
- 간편 회원 등록 폼 (이름, 성별, 생년월일, 연락처 등 필수 정보)
- 저장 후 상세 프로필 편집 화면으로 이동 가능

### [매칭생성] 바텀시트 — 오늘의 추천 프로필
- 최대 5개 프로필 카드 표시
- **데이터 부족 시에도 최소 3개 프로필은 표시** (추천 점수가 낮아도 노출)
- 프로필 카드 탭 → 상세 프로필 화면 (매칭 탭의 프로필 상세와 동일한 UI/매커니즘)
- 뒤로가기 → 바텀시트로 복귀

#### 추천 알고리즘 (점수 기반)
- **대상**: 현재 매니저 담당 회원 중 active 상태 & 미매칭 회원
- **점수 산출 항목** (6개, 동일 가중치):

| 항목 | 매칭 기준 | 점수 |
|---|---|---|
| 나이 | 상대와 ±5세 이내 | 0~100 (차이 적을수록 높음) |
| 학력 | 동등 이상 | 0 or 100 |
| 직업군 | 선호 직업군 매칭 여부 | 0 or 100 |
| 키 | 상대 희망 범위 내 | 0~100 |
| 종교 | 동일 또는 무관 | 0 or 100 |
| 연소득대 | 유사 범위 | 0~100 |

- 총점 = 6개 항목 평균 (0~100)
- 정렬: 총점 DESC → 최근 등록순
- **폴백**: 추천 대상이 3명 미만이면 최근 등록 active 회원으로 채움

### ~~[서류검토] 바텀시트~~ (MVP 제외)
> 회원 서류 검토 기능은 Post-MVP로 이동. 매니저 인증만 MVP에 포함.

## 1-4. 오늘의 할일

### 구성
- 카드 형태 리스트, 각 항목 탭 시 해당 화면으로 이동

| 항목 | 데이터 소스 | 탭 시 이동 |
|---|---|---|
| ⚠ 대기 매칭 N건 | `matches` WHERE status = 'pending' | 매칭 관리 바텀시트 |
| 💬 새 메시지 N건 | `messages` WHERE is_read = false | 채팅 탭 |

### 표시 규칙
- 숫자가 0이면 해당 항목 숨김 (할일이 없으면 섹션 자체 숨김)
- 실시간 업데이트: Supabase Realtime 구독으로 카운트 자동 갱신

### [대기 매칭 N건] 탭 → 매칭 관리 바텀시트

#### 구성
- 대기/진행/완료 상태별 탭 (상단)
- 각 매칭 항목: 회원A ↔ 회원B 카드, 매칭 상태, 생성일

#### 매칭 항목 구조
```
┌─────────────────────────────┐
│ 김서연 (29, 여) ↔ 이준호 (34, 남) │
│ 상태: 대기 중    2025.03.10      │
│ [수락] [거절] [메모]              │
└─────────────────────────────┘
```

#### 상태별 탭
| 탭 | 필터 | 표시 |
|---|---|---|
| 대기 | status = 'pending' | 수락/거절 버튼 |
| 진행 | status = 'accepted' / 'meeting_scheduled' | 상태 변경 옵션 |
| 완료 | status = 'completed' / 'declined' | 결과 확인만 |

#### 액션
- **수락**: status → 'accepted', 양측 매니저에게 알림
- **거절**: 거절 사유 입력 (선택) → status → 'declined'
- **메모**: 매칭 메모 추가/수정 (notes 컬럼)
- 매칭 항목 탭 → 매칭 상세 풀스크린 (양측 프로필 비교 뷰)

#### 데이터 소스
- `matches` 테이블 WHERE manager_id = current_user
- 정렬: matched_at DESC

## 1-5. 최근 활동 피드

### 구성
- 시간순 타임라인 (최신 → 오래된 순)
- **범위**: 최근 7일
- **최대 표시**: 20건 (스크롤로 더 불러오기 = 페이지네이션)
- 날짜 구분선: "오늘", "어제", "3월 9일" 등

### 표시할 이벤트 종류

| 이벤트 | 아이콘 | 예시 텍스트 |
|---|---|---|
| 매칭 생성 | 💕 | "김서연 ↔ 이준호 매칭 생성" |
| 매칭 수락 | ✅ | "김서연 ↔ 이준호 매칭 수락됨" |
| 매칭 거절 | ❌ | "김서연 ↔ 이준호 매칭 거절됨" |
| 회원 등록 | 👤 | "최민수 신규 회원 등록" |

### 각 피드 항목 구조
```
[아이콘] [시간] [내용 텍스트]
         [부가 설명 (선택)]
```

### 탭 동작
- 매칭 이벤트 탭 → 매칭 상세 화면
- 회원 이벤트 탭 → 회원 프로필 화면

### 데이터 소스
- `matches` 테이블 (매칭 이벤트)
- `clients` 테이블 (회원 등록 이벤트)
- 통합 쿼리: 2개 테이블 UNION → created_at DESC 정렬
- Supabase Realtime으로 실시간 업데이트 (새 이벤트 자동 추가)

### 빈 상태
- 활동이 없을 때: "아직 활동 내역이 없습니다. 첫 회원을 등록해보세요!" + 회원등록 버튼
